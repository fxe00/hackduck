# HackDuck 安装指南

## 开发环境安装

### 1. 安装依赖
```bash
npm install
```

### 2. 开发模式
```bash
npm run dev
```

### 3. 构建生产版本
```bash
npm run build
```

## 浏览器安装

### Chrome/Edge 安装

1. 打开Chrome浏览器，访问 `chrome://extensions/`
2. 开启右上角的"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择项目的 `dist` 文件夹
5. 安装完成后，打开任意网页，按F12打开开发者工具
6. 在开发者工具中会看到"HackDuck"标签页

### Firefox 安装

1. 打开Firefox浏览器，访问 `about:debugging`
2. 点击"此Firefox"
3. 点击"临时载入附加组件"
4. 选择项目的 `dist` 文件夹中的 `manifest.json` 文件
5. 安装完成后，打开任意网页，按F12打开开发者工具
6. 在开发者工具中会看到"HackDuck"标签页

## 使用说明

### 基本功能

1. **开启拦截**: 点击"拦截请求"开关开始捕获HTTP请求
2. **查看请求**: 在左侧列表中查看所有捕获的请求
3. **编辑请求**: 点击请求进入编辑模式，可以修改：
   - 请求方法 (GET, POST, PUT, DELETE等)
   - 请求URL
   - 请求头
   - 请求体
4. **发送请求**: 编辑完成后点击"发送请求"按钮
5. **过滤请求**: 使用搜索框过滤特定请求
6. **导出数据**: 点击"导出"按钮保存请求数据

### 高级功能

- **实时拦截**: 自动捕获页面发出的所有HTTP请求
- **请求重放**: 修改并重新发送任何请求
- **批量操作**: 支持批量修改和发送请求
- **数据导出**: 支持JSON格式导出请求数据

## 故障排除

### 常见问题

1. **扩展无法加载**
   - 确保已开启开发者模式
   - 检查manifest.json文件是否存在
   - 查看浏览器控制台错误信息

2. **无法拦截请求**
   - 确保已开启"拦截请求"开关
   - 检查页面是否使用了HTTPS
   - 某些请求可能被浏览器安全策略阻止

3. **开发者工具中看不到HackDuck**
   - 确保扩展已正确安装
   - 刷新页面后重新打开开发者工具
   - 检查扩展权限设置

### 调试模式

开启Chrome调试模式：
1. 右键点击扩展图标
2. 选择"检查弹出式窗口"
3. 查看控制台错误信息
