# HackDuck 测试指南

## 🚀 快速开始

### 1. 开发环境运行
```bash
# 启动开发模式（自动监听文件变化）
npm run dev

# 或者构建生产版本
npm run build
```

### 2. 浏览器安装扩展

#### Chrome/Edge 安装步骤：
1. 打开Chrome浏览器，访问 `chrome://extensions/`
2. 开启右上角的"开发者模式"开关
3. 点击"加载已解压的扩展程序"
4. 选择项目的 `dist` 文件夹
5. 确认安装成功（会看到HackDuck扩展图标）

#### Firefox 安装步骤：
1. 打开Firefox浏览器，访问 `about:debugging`
2. 点击"此Firefox"
3. 点击"临时载入附加组件"
4. 选择项目的 `dist` 文件夹中的 `manifest.json` 文件
5. 确认安装成功

### 3. 测试扩展功能

#### 基本测试流程：
1. **打开测试页面**：访问任意网站（如 https://httpbin.org）
2. **打开开发者工具**：按F12或右键选择"检查"
3. **查看HackDuck面板**：在开发者工具中会看到"HackDuck"标签页
4. **开启拦截**：在HackDuck面板中点击"拦截请求"开关
5. **触发请求**：在页面上进行一些操作（如点击按钮、提交表单等）
6. **查看捕获的请求**：在HackDuck面板左侧列表中查看捕获的HTTP请求

### 4. 常见问题排查

#### 问题1：扩展无法加载
**解决方案：**
- 确保已开启开发者模式
- 检查manifest.json文件是否存在
- 查看浏览器控制台错误信息

#### 问题2：开发者工具中看不到HackDuck
**解决方案：**
- 刷新页面后重新打开开发者工具
- 检查扩展是否正确安装
- 查看扩展权限设置

#### 问题3：无法拦截请求
**解决方案：**
- 确保已开启"拦截请求"开关
- 检查页面是否使用HTTPS
- 某些请求可能被浏览器安全策略阻止

#### 问题4：请求编辑后发送失败
**解决方案：**
- 检查URL格式是否正确
- 检查请求头是否有效
- 检查请求体格式是否正确
