# HackDuck 项目总结

## 🎯 项目概述

HackDuck 是一个基于Vue 3 + Element Plus的浏览器扩展，专门用于HTTP请求的调试和管理。当开发者按下F12打开调试工具时，会出现HackDuck工具栏目，提供强大的HTTP请求拦截、编辑和发送功能。

## 🏗️ 技术架构

### 前端技术栈
- **Vue 3** - 现代化的前端框架
- **Element Plus** - 企业级UI组件库
- **TypeScript** - 类型安全的JavaScript
- **Webpack** - 模块打包工具

### 浏览器扩展技术
- **WebExtensions API** - 跨浏览器扩展标准
- **Chrome DevTools API** - 开发者工具集成
- **WebRequest API** - HTTP请求拦截
- **Content Scripts** - 页面脚本注入

## 🚀 核心功能

### 1. 请求拦截
- 实时捕获浏览器发出的所有HTTP请求
- 支持GET、POST、PUT、DELETE等所有HTTP方法
- 自动记录请求时间、状态码、响应时间等信息

### 2. 请求编辑
- **基本信息编辑**: 修改请求方法、URL
- **请求头编辑**: 添加、删除、修改HTTP头
- **请求体编辑**: 支持JSON、XML、表单数据等格式
- **实时预览**: 编辑时实时显示请求内容

### 3. 请求发送
- 一键发送修改后的请求
- 支持跨域请求
- 实时显示响应结果
- 错误处理和状态提示

### 4. 数据管理
- **请求历史**: 保存所有拦截的请求
- **数据过滤**: 按URL、方法、状态码过滤
- **数据导出**: 支持JSON格式导出
- **批量操作**: 支持批量修改和发送

## 🔧 开发流程

### 1. 环境准备
```bash
# 安装Node.js和npm
# 克隆项目
git clone <repository>
cd hackduck

# 安装依赖
npm install
```

### 2. 开发模式
```bash
# 启动开发服务器
npm run dev

# 或使用启动脚本
./start.sh
```

### 3. 构建部署
```bash
# 构建生产版本
npm run build

# 打包扩展
npm run package
```

## 📦 安装使用

### 开发者安装
1. 构建项目: `npm run build`
2. 打开Chrome: `chrome://extensions/`
3. 开启开发者模式
4. 加载`dist`文件夹
5. 按F12打开开发者工具
6. 查看HackDuck标签页

### 用户安装
1. 下载扩展包
2. 拖拽到浏览器扩展页面
3. 确认安装
4. 开始使用
